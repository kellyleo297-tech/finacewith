规则层 + LLM 两层决策

Supervisor 不是每次都调 LLM。关键词组合匹配覆盖约 70% 高频场景，0ms 延迟、0 token。未命中才走 LLM。

System Prompt 5 要素模板

角色定义 → 能力边界 → 数据注入 → JSON 输出约束 → 安全规则。温度参数按 Agent 职责分层（0.1~0.5）。

03 · 关键产品决策

每一个设计选择
背后都有明确理由

"代码算账，Agent 解释"

金额计算、预算判断、图表数据全部由代码完成。Agent 只引用数据库查询注入的真实数据做自然语言解释。杜绝 AI 幻觉导致财务数据错误。

AI 记账不自动写入，弹确认卡片

AI 识别不是 100% 准确的。如果分类错了自动写入 → 用户不知情 → 统计数据污染。点击确认的成本很低，但避免静默错误的成本很高。这是有意为之的"摩擦设计"。

首页只回答一个问题："今天还能花多少？"

用户打开记账 App 的心理状态不是"我想看财务全景"，而是"我现在还能花多少"。剩余预算 ÷ 剩余天数 = 今日可花。

7 Agent 架构 vs 一个全能 Agent

多 Agent = 每个可独立评测、独立优化。风控 Agent 和其他 Agent 代码零耦合。一个 Agent 出问题，其他不受影响。

04 · 风控设计

两层防御
一个原则

财务 AI 最大的风险是用户把它当投资顾问。
我们的策略：不允许它变成投资顾问。

🛡️ 预防层（Prompt 层）

理财 Agent 的 System Prompt 内置 7 条红线：不推荐股票/基金代码、不承诺收益、不诱导借钱投资、不替用户做投资决策。

🚨 拦截层（风控 Agent）

理财 Agent 输出 → 风控 Agent 二次扫描。红线违规直接修正，黄线问题补充风险提示。风控 Agent 与其他 Agent 零代码耦合。
